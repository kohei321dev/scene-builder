# Product Brief: Scene Builder

## Name

採用名は `Scene Builder`。repo名は `scene-builder`。

理由:

- シチュエーション別に英文を組み立てる、というプロダクトの中心を表している
- スケボー専用ではないため、日常会話、旅行、仕事、友達との雑談にも拡張できる
- 個人学習用でも、URLを知っている人が使える小さな学習サイトとしても違和感が少ない
- GitHub repository名として短く扱いやすい

## Problem

[事実] 学習者は英検3級程度の基礎文法と暗記済みの短い英文を持っているが、自分が言いたい内容をその場で英文に組み立てる力に課題がある。

[推測] スピーキングが詰まる主因は発音や瞬発力だけではなく、話す前の短文ライティング能力、語彙選択、基本語順の自動化が不足していること。

## Goal

スケボー中の自然な日常会話で、以下ができる状態を目指す。

- 自分の経験を短く説明する
- 相手の道具、技、練習内容について質問する
- ほめる、誘う、断る、もう一度聞く
- 失敗、練習、体調、予定を簡単に伝える
- 1文から2文で理由や感想を追加する

## Learning Methods

### Topic Cards

IELTS/TOEFL系のトピックカードを、日常会話とスケボー場面に寄せて使う。

- 場面を絞る
- 回答の型を増やす
- 同じカードに難易度別の回答を持たせる
- 回答後に使える語彙を蓄積する

### Short Diary

毎日または練習後に、短い英語日記を書く。

- 3文以内から始める
- 今日やったこと、できたこと、できなかったことを書く
- 同じ内容を難易度別に言い換える
- スピーキングで使う前提の文にする

## Difficulty Ladder

「動詞だけの英文」は自然な英語としては成立しにくいため、ここでは「動詞中心で、修飾語をほぼ使わない回答」を最低難易度として扱う。

| Level | Name | Target |
| --- | --- | --- |
| L1 | Verb focus | 主語 + 動詞 + 目的語/場所。形容詞や副詞をほぼ使わない |
| L2 | Add detail | 形容詞を1つ足す。気持ち、場所、物の状態を少し言う |
| L3 | Reason | because / so / but を使い、理由や対比を足す |
| L4 | Conversation | 相手への質問、誘い、確認を足して会話にする |

## MVP

### MVP 0: Data First

- CSVでトピックカードを作る
- CSVで日記プロンプトを作る
- 難易度別の模範回答を蓄積する
- ChatGPTに貼れる添削プロンプトを作る

### MVP 1: Static Web App

- カテゴリ別にカードを表示する
- 難易度を切り替える
- 自分の回答をブラウザ内で入力する
- 模範回答とセルフチェック項目を表示する
- 学習済みカードをlocalStorageに保存する

### MVP 2: AI Review

- Vercel FunctionsなどでAI APIを呼ぶ
- ユーザー回答を採点する
- 自然な言い換えを返す
- 追加練習カードを生成する
- 初期は認証済みユーザーだけがAI Reviewを使えるようにする

## Feasibility

[事実] Vercel上の静的サイトでも、カード表示、難易度切替、localStorageによる学習状態保存は実装できる。

[事実] ユーザー入力に対する意味理解、文法添削、自然な言い換え提案には、AI APIまたは外部サービスが必要になる。

[推測] 最初からAI採点を入れるより、静的カードとセルフチェックで学習データを作り、使い続けられるか確認した後にAI採点を追加する方が実装コストと運用コストを抑えられる。

[事実] チャットアプリのサブスクリプションとAPI利用料金は、主要providerで別扱いになることが多い。API連携では、API key、server-side実行、利用上限、ログ設計を分けて扱う。

## Non Goals

- 初期段階では本格的な発音評価をしない
- 初期段階では大学入試、IELTS、TOEFL向けのアカデミックライティングを主目的にしない
- MVP 0ではユーザーアカウントやクラウドDBを持たない
